## What's Changed

### Added
- Comic basic info system with local DB caching (title, subtitle, cover, description, tags) shared between app and web
- Web↔App WebDAV sync for backup/restore data portability
- Favorites parity across web PWA and Flutter app
- Native-aligned Vue 3 web frontend with full feature parity
- Mirror comic metadata to domain DB from history, favorites, search results, and local import
- Server backfill of comic basic info from history for initial data migration
- Multi-platform sync format spec with badges, backup naming conventions, and sync logs
- Loading spinner on comic detail page while network fetch is in progress
- Per-tab pull-to-refresh on explore page
- Windows updater downloads update package with progress dialog and cancel support

### Changed
- Comic detail page returns placeholder data immediately and fetches network details in background
- Re-enrich comics with local info BEFORE setting reactive data (eliminates UI flash)
- Restructure web workspace into `web/client/` namespace
- Web reader improvements and mobile layout fixes
- Docker restructure for web PWA deployment
- App version read from `package_info_plus` instead of hardcoded
- `checkUpdateOnStart` defaults to `true`

### Fixed
- DB queries before initialization now return empty results instead of crashing
- `computeImageFavorites` wrapped in try-catch to prevent Isolate crashes
- Comic chapter grouping for source scripts with broken version comparison
- Skip empty comic source duplicates in web
- WebDAV server DB parity helpers
- Web helper API route parity with app
- Web frontend server DB contracts
- Web sync and follow updates parity
- Explore page merge cached comic basic info server-side in explore response

### Removed
- Legacy web PWA

### Build / CI
- Revert tsconfig `ignoreDeprecations` (incompatible with Docker TS version)
- Add `package_info_plus` dependency
